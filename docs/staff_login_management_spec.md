# NETSHOP 担当者ログイン管理 仕様書

対象日: 2026-03-25  
対象システム: NETSHOP Web画面 / GAS / スプレッドシート運用

## 目的

- 現行稼働中の本番システムの挙動は変更せず、今後の開発運用ルールと実装先を整理する
- 既存の公開URL体系を維持したまま、開発確認と本番確認の運用を明確化する
- 実装責務を親GASプロジェクトへ集約し、今後の保守性を上げる
- 出品担当者の手入力を廃止し、ログイン後に選択した担当者を基準に自動反映する
- 利用者の IT スキルが低くても迷わず使える運用にする
- 担当者ごとの検索、集計、管理をしやすくする
- 将来の権限管理や複数アカウント運用へ拡張しやすくする

## 基本方針

| 項目 | 方針 |
|---|---|
| 現行本番 | 現在の本番システム自体は触らない |
| 既存URL | 既存URLは維持する |
| 実装先 | 実装は親GASプロジェクト側で行う |
| ライブラリ | GASライブラリは使わない |
| 認証 | Googleアカウントでログインする |
| 利用者数 | 初期は Googleアカウント 1 つで運用する |
| 担当者管理 | ログイン後に担当者を選択して確定する |
| 担当者採番 | 担当者設定時に 担当者ID を自動採番する |
| 商品登録 | 出品担当者 は手入力させず自動反映する |
| 管理単位 | 担当者ID + 担当者名 で管理する |
| 検索 | 担当者名だけでなく担当者IDでも検索できるようにする |
| 操作性 | ログイン後に担当者を選ぶだけで登録できる状態にする |

## 運用ルール

| 区分 | ルール |
|---|---|
| 開発確認 | /dev URL を使用する |
| 本番確認 | /exec URL を使用する |
| URL運用 | 既存URLは維持し、開発確認は同じ /dev URL で継続する |
| 実装対象 | 親GASプロジェクトのみを変更対象とする |
| 配置方針 | doGet / HTML / API / UIロジックは親GASプロジェクトに配置する |
| ライブラリ方針 | GASライブラリ前提の新規実装は行わない |
| デプロイ更新 | 開発中は deployment を更新しない |
| 本番反映 | 本番反映時のみ version 作成と deployment 更新を行う |
| clasp運用 | clasp push 後は同一の /dev URL で更新確認を行う |
| /exec 更新 | /exec URL は最終反映時のみ更新する |
| 出品担当者 | 商品登録画面で手入力しない |
| 担当者確定 | ログイン後に選択した担当者情報を利用する |

## 変更制約

| 区分 | 内容 |
|---|---|
| 禁止事項 | 現行稼働中の本番挙動を変える変更は行わない |
| 禁止事項 | 今回の目的に不要な構造変更は行わない |
| 禁止事項 | ライブラリ依存を前提にした新規実装は行わない |
| 許容範囲 | 今後の開発運用ルール整備と実装先の整理に限定する |

## 開発・反映フロー

1. 実装は親GASプロジェクト上で行う
2. 開発中は clasp push により親プロジェクトへ反映する
3. 開発確認は既存の /dev URL で行う
4. 開発段階では version 作成と deployment 更新を行わない
5. 本番反映の最終段階でのみ version を作成する
6. 本番反映の最終段階でのみ /exec 向け deployment を更新する

## 想定ユーザーフロー

1. 利用者が Google アカウントで Web 画面を開く
2. システムが担当者選択画面を表示する
3. 利用者が担当者マスタから自分の担当者を選択する
4. 選択した担当者IDと担当者名をセッションへ保持する
5. 担当者未登録なら担当者追加画面へ進む
6. 登録時に担当者IDを自動採番する
7. 保存後は担当者ホームへ遷移する
8. 以後の商品登録では 担当者ID + 担当者名 を自動反映する

## 担当者マスタ仕様

### 管理目的

- 業務上の担当者を一意に管理する
- 表記ゆれのない担当者管理を行う
- 将来の権限や利用停止にも対応できる構造にする

### 推奨項目

| 項目 | 内容 | 必須 | 備考 |
|---|---|---|---|
| 担当者ID | システム採番ID | 必須 | ST-0001 形式固定 |
| 担当者名 | 画面表示用名称 | 必須 | 一覧、詳細、検索で使用 |
| 表示順 | 並び順制御 | 任意 | 管理画面用 |
| 利用状態 | 有効 / 無効 | 必須 | 無効時は選択不可 |
| 備考 | 管理メモ | 任意 | 管理者用 |
| 登録日 | 登録日時 | 必須 | 自動記録 |
| 更新日 | 更新日時 | 必須 | 自動記録 |

### 採番ルール

| 項目 | 仕様 |
|---|---|
| 採番タイミング | 初回担当者登録時 |
| 採番方法 | 最大番号 + 1 |
| 形式 | ST-0001 形式で固定 |
| 重複防止 | 保存前に担当者IDと担当者名の重複確認を行う |

### 初期運用

| 担当別表示 | 一覧初期表示を自担当のみへ絞り込み |
| 承認フロー | 必要時に管理者承認を追加 |

## Web画面仕様

### 1. ログイン後判定

| 条件 | 画面動作 |
|---|---|
| 担当者選択済み | そのまま担当者ホームへ進む |
| 担当者未選択 | 担当者選択画面へ進む |
| 担当者未登録 | 担当者追加画面へ進む |
| 利用状態が無効 | 利用不可メッセージを表示する |

### 2. 担当者選択画面

| 項目 | 仕様 |
|---|---|
| 目的 | 業務上の担当者を確定する |
| 表示内容 | 担当者一覧 |
| 操作 | 自分の担当者を選択する |
| 保存後 | 担当者ホームへ遷移 |
| 補足 | 画面上で現在の担当者を常時表示する |

### 3. 担当者追加画面

| 項目 | 仕様 |
|---|---|
| 目的 | 初回利用者や新担当者の登録 |
| 入力項目 | 担当者名 |
| 自動採番 | 担当者ID |
| 登録権限 | 全員登録可 |
| 保存後 | 担当者選択済みとして担当者ホームへ遷移 |
| 補足 | 担当者ホームからも追加できるようにする |

### 4. 担当者ホーム画面

| 項目 | 仕様 |
|---|---|
| 表示位置 | ログイン後の起点画面 |
| 表示内容 | 担当者ID / 担当者名 |
| 主目的 | 現在選択中の担当者確認 |
| 操作 | 商品一覧 / 新規登録 / ダッシュボードへ進める |
| 補足 | 担当者切替 と 担当者追加 ボタンを設置できる構成にする |

### 5. 新規商品登録画面

| 項目 | 仕様 |
|---|---|
| 出品担当者 | 手入力しない |
| 表示内容 | 担当者IDと担当者名を表示 |
| 保存値 | シート列 C: 出品担当者 へ反映 |
| 推奨形式 | ST-0001 山田太郎 など、ID + 名前で保存 |

## スプレッドシート反映仕様

| 列 | 項目 | 反映内容 |
|---|---|---|
| C | 出品担当者 | 担当者ID + 担当者名 を自動反映 |

### 保存形式の考え方

- 運用利便性を優先し、シートには一目で分かる表示を保存する
- 検索性を上げるため、担当者ID と 担当者名 を同時に持つ形式にする

例:

`	ext
ST-0001 山田太郎
`

必要に応じて内部データでは次を保持する。

- staffId
- staffName

## エラー時の扱い

| 条件 | 対応 |
|---|---|
| 担当者未選択 | 担当者選択画面へ戻す |
| 担当者未登録 | 担当者追加画面へ誘導 |
| 担当者が無効 | 利用停止メッセージを表示 |
| 担当者重複 | 管理者確認を促すエラーを表示 |
