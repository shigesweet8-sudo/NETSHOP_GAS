# NETSHOP 担当者ログイン管理 仕様書

対象日: 2026-03-25  
対象システム: NETSHOP Web画面 / GAS / スプレッドシート運用

## 目的

- 出品担当者の手入力を廃止し、ログイン後に選択した担当者を基準に自動反映する
- 利用者の IT スキルが低くても迷わず使える運用にする
- 担当者ごとの検索、集計、管理をしやすくする
- 将来の権限管理や複数アカウント運用へ拡張しやすくする

## 基本方針

| 項目 | 方針 |
|---|---|
| 認証 | Googleアカウントでログインする |
| 利用者数 | 初期は Googleアカウント 1 つで運用する |
| 担当者管理 | ログイン後に担当者を選択して確定する |
| 担当者採番 | 担当者設定時に `担当者ID` を自動採番する |
| 商品登録 | `出品担当者` は手入力させず自動反映する |
| 管理単位 | `担当者ID + 担当者名` で管理する |
| 検索 | 担当者名だけでなく担当者IDでも検索できるようにする |
| 操作性 | ログイン後に担当者を選ぶだけで登録できる状態にする |

## 運用ルール

| 区分 | ルール |
|---|---|
| 開発確認 | `dev` URL を使用する |
| 固定版確認 | `exec` URL を使用する |
| 旧URL | `AKfycbzgi...` 系 URL は使用しない |
| 出品担当者 | 商品登録画面で手入力しない |
| 担当者確定 | ログイン後に選択した担当者情報を利用する |

## 想定ユーザーフロー

1. 利用者が Google アカウントで Web 画面を開く
2. システムが担当者選択画面を表示する
3. 利用者が担当者マスタから自分の担当者を選択する
4. 選択した担当者IDと担当者名をセッションへ保持する
5. 担当者未登録なら担当者追加画面へ進む
6. 登録時に担当者IDを自動採番する
7. 保存後は担当者ホームへ遷移する
8. 以後の商品登録では `担当者ID + 担当者名` を自動反映する

## 担当者マスタ仕様

### 管理目的

- 業務上の担当者を一意に管理する
- 表記ゆれのない担当者管理を行う
- 将来の権限や利用停止にも対応できる構造にする

### 推奨項目

| 項目 | 内容 | 必須 | 備考 |
|---|---|---|---|
| 担当者ID | システム採番ID | 必須 | `ST-0001` 形式固定 |
| 担当者名 | 画面表示用名称 | 必須 | 一覧、詳細、検索で使用 |
| 表示順 | 並び順制御 | 任意 | 管理画面用 |
| 利用状態 | 有効 / 無効 | 必須 | 無効時は選択不可 |
| 備考 | 管理メモ | 任意 | 管理者用 |
| 登録日 | 登録日時 | 必須 | 自動記録 |
| 更新日 | 更新日時 | 必須 | 自動記録 |

### 採番ルール

| 項目 | 仕様 |
|---|---|
| 採番タイミング | 初回担当者登録時 |
| 採番方法 | 最大番号 + 1 |
| 形式 | `ST-0001` 形式で固定 |
| 重複防止 | 保存前に担当者IDと担当者名の重複確認を行う |

## ログイン管理仕様

### 初期運用

| 項目 | 仕様 |
|---|---|
| ログイン方式 | Googleアカウント 1 つで開始 |
| 対象者 | 複数担当者でも共通アカウント運用可 |
| セキュリティ | Apps Script 実行ユーザーの認証を利用 |
| 初回動作 | 担当者が未登録なら担当者追加画面へ誘導 |

### 将来拡張

| 項目 | 拡張内容 |
|---|---|
| 複数担当者 | ログイン後の担当者選択候補を増やす |
| 権限管理 | 管理者 / 一般担当 / 閲覧のみ |
| 担当別表示 | 一覧初期表示を自担当のみへ絞り込み |
| 承認フロー | 必要時に管理者承認を追加 |

## Web画面仕様

### 1. ログイン後判定

| 条件 | 画面動作 |
|---|---|
| 担当者選択済み | そのまま担当者ホームへ進む |
| 担当者未選択 | 担当者選択画面へ進む |
| 担当者未登録 | 担当者追加画面へ進む |
| 利用状態が無効 | 利用不可メッセージを表示する |

### 2. 担当者選択画面

| 項目 | 仕様 |
|---|---|
| 目的 | 業務上の担当者を確定する |
| 表示内容 | 担当者一覧 |
| 操作 | 自分の担当者を選択する |
| 保存後 | 担当者ホームへ遷移 |
| 補足 | 画面上で現在の担当者を常時表示する |

### 3. 担当者追加画面

| 項目 | 仕様 |
|---|---|
| 目的 | 初回利用者や新担当者の登録 |
| 入力項目 | 担当者名 |
| 自動採番 | 担当者ID |
| 登録権限 | 全員登録可 |
| 保存後 | 担当者選択済みとして担当者ホームへ遷移 |
| 補足 | 担当者ホームからも追加できるようにする |

### 4. 担当者ホーム画面

| 項目 | 仕様 |
|---|---|
| 表示位置 | ログイン後の起点画面 |
| 表示内容 | `担当者ID / 担当者名` |
| 主目的 | 現在選択中の担当者確認 |
| 操作 | 商品一覧 / 新規登録 / ダッシュボードへ進める |
| 補足 | `担当者切替` と `担当者追加` ボタンを設置できる構成にする |

### 5. 新規商品登録画面

| 項目 | 仕様 |
|---|---|
| `出品担当者` | 手入力しない |
| 表示内容 | 担当者IDと担当者名を表示 |
| 保存値 | シート列 `C: 出品担当者` へ反映 |
| 推奨形式 | `ST-0001 山田太郎` など、ID + 名前で保存 |

## スプレッドシート反映仕様

| 列 | 項目 | 反映内容 |
|---|---|---|
| C | 出品担当者 | `担当者ID + 担当者名` を自動反映 |

### 保存形式の考え方

- 運用利便性を優先し、シートには一目で分かる表示を保存する
- 検索性を上げるため、`担当者ID` と `担当者名` を同時に持つ形式にする

例:

```text
ST-0001 山田太郎
```

必要に応じて内部データでは次を保持する。

- `staffId`
- `staffName`

## 検索・一覧・管理仕様

| 項目 | 仕様 |
|---|---|
| 一覧表示 | 担当者列は `担当者ID + 担当者名` を表示 |
| キーワード検索 | 担当者IDでも担当者名でも一致するようにする |
| 絞り込み拡張 | 将来は担当者別フィルタを追加可能にする |
| 詳細画面 | 担当者情報を表示する |
| ダッシュボード | 将来、担当者別集計へ拡張できる構造にする |

## エラー時の扱い

| 条件 | 対応 |
|---|---|
| 担当者未選択 | 担当者選択画面へ戻す |
| 担当者未登録 | 担当者追加画面へ誘導 |
| 担当者が無効 | 利用停止メッセージを表示 |
| 担当者重複 | 管理者確認を促すエラーを表示 |

## 利用者配慮

| 項目 | 配慮内容 |
|---|---|
| 入力削減 | 担当者名の手入力をなくす |
| 迷い防止 | ログイン後に担当者を選ぶだけにする |
| 表示簡素化 | 難しい用語を避ける |
| エラー案内 | 次に何をすればよいか明記する |
| 誤登録防止 | 選択中担当者を画面上部に常時表示する |
| 現場運用 | 担当者ホームから担当者追加できるようにする |

## 決定事項

| 項目 | 決定内容 |
|---|---|
| 担当者入力方式 | 手入力ではなくログイン管理ベース |
| 担当者採番 | 初回登録時に自動採番 |
| 担当者ID形式 | `ST-0001` 形式で固定 |
| シート保存形式 | `ID + 名前` の1列保存 |
| 管理単位 | 担当者ID + 担当者名 |
| 検索単位 | 担当者ID + 担当者名 |
| 初期運用 | Googleアカウント 1 つで開始し、ログイン後に担当者選択 |
| ホーム名称 | `担当者ホーム` とする |
| 初回設定権限 | 全員登録可 |
| 追加導線 | 担当者ホームから担当者追加可能にする |
| 重視点 | 運用利便性と入力ミス防止 |
