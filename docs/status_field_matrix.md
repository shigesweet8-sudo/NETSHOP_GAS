## 共通ルール

| 区分 | 内容 |
|---|---|
| 基準 | シート列と `CONFIG.COLS` / `CONFIG.HEADERS` を基準にする |
| 自動反映 | B は保存時自動採番、D と E は新規登録画面で当日を自動反映 |
| 郵便番号 | V 入力後、住所補完で W・X を自動反映 |
| 未入力警告 | ステータス別の必須扱い項目が未入力なら赤ハイライト表示 |
| 保存可否 | 未入力があっても保存・ステータス移行は可能 |
| 完了系表示 | `発送完了` / `取引完了` は完了系塗りつぶし、`キャンセル` はクローズ系塗りつぶし |

## 詳細ルール

| ステータス | 推奨項目 | 説明 |
|---|---|---|
| 商品登録 | R | 備考がある場合に記録 |
| 出品中 | R | 出品条件の補足を記録 |
| 値段交渉中 | R | 交渉経緯のメモを残す |
| 決済完了 | R | 決済メモや補足を残す |
| 発送準備中 | R | 発送時の注意事項を残す |
| 発送完了 | U, R | 追跡番号とメモがあると確認しやすい |
| 取引完了 | R | 完了記録用メモ |
| キャンセル | R | キャンセル理由の記録推奨 |

## システム反映メモ

| 区分 | 現在の反映内容 |
|---|---|
| 新規登録画面 | ステータス変更に応じて入力計画を表示 |
| ハイライト | 必須扱い項目を黄色、未入力項目を赤で表示 |
| 自動日付 | D, E に当日を初期表示 |
| 管理ID | B は保存前プレビューのみ、保存時に採番 |
| 住所補完 | V 入力後に住所補完ボタンで W, X を反映 |
| 保存前確認 | 必須不足は warning として表示、保存は禁止しない |

## 運用上の注意

| 項目 | 内容 |
|---|---|
| ステータス名称 | コード上は `発送準備中` / `発送完了` を使用する |
| 列指定方法 | 今後の指示は列記号またはヘッダー名で指定可能 |
| 未実装判断 | 新たな表示条件や非表示条件はこの表を基準に追加整理する |
