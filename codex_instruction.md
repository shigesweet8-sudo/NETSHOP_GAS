## 目的
`getMasters` / `api_getMasters` の参照元を固定し、ステータス・ショップ・担当者・配送元・配送業者について、シート現存値や入力規則に依存しない安定した正式マスター一覧を返すように実装する。

## 実装スコープ
- `getMasters` が返す各マスターの参照元を固定化する。
- 対象は以下の5種に限定する。
  - ステータス
  - ショップ
  - 担当者
  - 配送元
  - 配送業者
- `api_getMasters` から返るレスポンスも上記の正式マスターを使用する。
- 未定義のマスター種別は安全に空配列を返す。
- 既存 API 基盤や他 API のレスポンス仕様は壊さない。
- 必要に応じて `01_config.gs` に正式マスター定義を追加する。

## 対象ファイル
- `20_api_netshop.gs`
- 必要に応じて `01_config.gs`

候補となる実装箇所:
- `getMasters`
- `api_getMasters`
- マスター参照用の内部関数（新規追加可）
- 固定マスター定義の定数群（`01_config.gs` への追加を優先）

## 制約
- 実装 Issue なので、実際の `.gs` 実装ファイルを変更すること。
- 返却値は Web UI でそのまま使える形式を維持すること。
- 業務上の正式候補と一致する形で定義すること。
- シート未使用の候補でも、正式マスターに含まれるものは返せる構成にすること。
- 未定義マスターは例外にせず空配列を返すこと。
- 既存のシート入力規則取得や現存データ走査に依存する処理が `getMasters` の正式参照元として残らないようにすること。
- push、ブランチ操作、ドキュメント更新のみで終えることは禁止。
- 情報不足がある場合は、既存コード内の現在の候補値・定数・列定義を優先して正式マスター定義へ寄せること。

## 非対象
- UI 実装
- マスター画面の新規作成
- 他 API の仕様変更
- この Issue と無関係なリファクタ
- `codex_instruction.md` / `issue_tasks.md` / `openai_response.json` のみを変更して完了とすること

## 実装ステップ
1. `20_api_netshop.gs` 内の `getMasters` / `api_getMasters` の現在の参照元を確認し、シート現存値や入力規則から直接マスターを組み立てている箇所を、固定マスター参照へ置き換える。
2. `01_config.gs` に、対象5種の正式マスター定義を追加する。既に近い定数がある場合は再利用し、なければ `NETSHOP_*` など既存命名に合わせた定数を新設する。
3. `20_api_netshop.gs` に、マスター種別ごとに正式定義を返す内部関数を追加または整理する。  
   - 対象種別: `status`, `shop`, `assignee`, `shippingFrom`, `carrier` 相当  
   - 未定義キーは `[]` を返す
4. `getMasters` が常に上記内部関数経由で正式マスターを返すように修正する。
5. `api_getMasters` のレスポンスが既存形式を維持しつつ、正式マスターを返すことを確認し、必要ならキー名マッピングを既存 API 仕様に合わせて調整する。
6. 既存の入力規則・現存データ依存の補助関数が `getMasters` から不要になった場合でも、この Issue の範囲で安全に扱うこと。削除がリスク高なら未使用化に留めてよい。
7. 少なくとも `getMasters` / `api_getMasters` の呼び出しで、対象5種が配列で返り、未定義種別が空配列になることを確認する。

## 完了条件
- `20_api_netshop.gs` に実コード差分があること。
- 必要に応じて `01_config.gs` に正式マスター定義が追加されていること。
- `getMasters` / `api_getMasters` が、ステータス・ショップ・担当者・配送元・配送業者を固定参照元から返すこと。
- シート現存値や入力規則への依存が、少なくとも `getMasters` の正式参照元としては除去されていること。
- 未定義マスターは空配列で返ること。
- `md/json/txt` だけの変更で終わっていないこと。

## 検証
- Apps Script 上で構文エラーがないことを確認する。
- `getMasters()` を実行し、対象5種のマスターが安定した配列で返ることを確認する。
- `api_getMasters()` を実行し、既存 API 形式を壊さずに正式マスターが返ることを確認する。
- シートに存在しない候補でも、正式マスター定義に含まれる値が返ることを確認する。
- 未定義のマスター種別指定がある実装なら、そのケースで `[]` が返ることを確認する。
