## 目的
`20_api_netshop.gs` に、商品一覧を指定条件で並び替えて返す API を追加する。既存の `listItems` と整合したデータ取得を維持しつつ、レスポンス上の順序のみを変更できるように実装する。

## 実装スコープ
- 並び替え用の関数を `20_api_netshop.gs` に追加する
- API 入口を `20_api_netshop.gs` に追加する
- 少なくとも以下を扱う
  - 並び替えキー指定
  - 昇順 / 降順指定
- 既存の `listItems` と整合すること
- シートの行順は変更せず、返却配列の順序だけを変更すること
- 並び替え未指定時は既存順を維持すること
- 存在しないキー指定は安全に扱うこと（既存順維持またはエラー応答のいずれかで、既存 API 基盤を壊さない実装にする）

## 対象ファイル
- 必須:
  - `20_api_netshop.gs`
- 必要に応じて参照のみ:
  - 既存 API のルーティングやレスポンス形式が分かる他の `.gs` ファイル
- ただし、実コード変更は少なくとも `20_api_netshop.gs` に必ず入れること

## 制約
- `doPost` は新規追加しない
- 既存 API 基盤を壊さないこと
- `listItems` の取得仕様・返却形式との整合を優先すること
- 並び替えはシート更新ではなくメモリ上の配列操作で実装すること
- キー未指定時は既存順維持
- 存在しないキー指定時の挙動は実装内で一貫させること
- 情報不足がある場合は、既存の API 入口実装方式・パラメータ取得方式・レスポンス生成方式に合わせて推測実装すること
- `codex_instruction.md` / `issue_tasks.md` / `openai_response.json` だけを変更して終えてはいけない

## 非対象
- UI 実装
- `[15] キャンセル処理API`
- `[16] 一括削除 / 無効化API`
- 既存シート構造の変更
- 永続データの並び順変更
- `doPost` の新規実装
- ドキュメントのみの変更

## 実装ステップ
1. `20_api_netshop.gs` の既存 `listItems` 実装、API 入口、レスポンス形式、パラメータ取得方法を確認し、それに合わせた追加方針にする。
2. `listItems` と同等の一覧データを利用し、配列を破壊しすぎない形でソート済み配列を返す並び替え関数を追加する。
   - 比較対象キーを引数で受け取る
   - 昇順 / 降順を引数で受け取る
   - 未指定時は元順を維持する
   - 存在しないキーは安全に扱う
   - 安定した既存順維持が必要なため、元インデックスを保持して同値時の順序を維持する実装を優先する
3. API 入口を追加する。
   - 既存ルーティング方式に従って新しいアクション/メソッド名を追加する
   - リクエストから並び替えキー・昇降順を受け取る
   - 既存 `listItems` と整合した一覧結果をベースに、ソート後レスポンスを返す
4. エラー処理または無視処理を既存レスポンス形式に合わせて実装する。
   - 不正なキーや不正な order 値が来た際に API 基盤を壊さないこと
   - 可能なら `asc` / `desc` を正規化し、それ以外は既定値にフォールバックする
5. 既存コードとの重複が大きい場合は、`listItems` の内部共通化を最小限で行ってよい。ただし別機能を混ぜず、この Issue の範囲内に限定する。
6. 必要なら軽微なログやガードを追加し、既存 API から呼ばれた際に例外で全体が落ちにくいようにする。

## 完了条件
- `20_api_netshop.gs` に実コード差分がある
- 並び替え API 本体が追加されている
- API 入口が追加されている
- 並び替えキー指定と昇順 / 降順指定に対応している
- シートの行順を変更せず、レスポンスの配列順のみ変更している
- 未指定時は既存順を維持している
- 存在しないキー指定時も安全に動作する
- `md/json` だけの変更ではなく、実装ファイル変更で完了している

## 検証
- 既存の一覧取得と同じ条件で、並び替え指定なしのレスポンス順が従来と一致することを確認する
- 並び替えキーを指定したときに昇順で並ぶことを確認する
- 同じキーで `desc` 指定時に降順で並ぶことを確認する
- 存在しないキー指定時に、例外で API 全体が壊れず、安全な応答になることを確認する
- シート上のデータ順が変更されていないことを確認する
- 既存の他 API ルーティングに影響が出ていないことを確認する
