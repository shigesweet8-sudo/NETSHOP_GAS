## 背景
NETSHOP GAS の API 基盤復旧後、Phase1 を正式番号順で再開します。
今回は [6] 一括ステータス更新API を実装対象とします。

## 正式番号
- [6] 一括ステータス更新API

## 実装対象ファイル
- `20_api_netshop.gs`

## 目的
複数の管理IDに対して、同じステータスを一括更新できる API を追加する。

## 必須実装
- 複数の管理IDを受け取り、同一ステータスへ一括更新する関数を追加する
- GAS の既存呼び出し方式に合わせた API 入口を追加する
- 成功件数、失敗件数、更新成功ID一覧、失敗ID一覧を返す
- 不正なステータス値は更新せずエラー扱いにする
- 存在しない管理IDは失敗として返す
- `doPost` は新規追加しない

## 入力例
```json
{
  "itemIds": ["CP-A0001", "CP-A0002"],
  "status": "発送準備中",
  "memo": "一括更新"
