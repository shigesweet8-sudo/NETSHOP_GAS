## 背景
NETSHOP GAS の Web化 Phase1 を進めます。
[12] プラットフォーム別集計API が完了したため、次は [17] マスター取得API を実装対象とします。

## 正式番号
- [17] マスター取得API

## 実装対象ファイル
- `20_api_netshop.gs`

## 目的
Web UI で使う各種マスター値を API で取得できるようにする。

## 必須実装
- マスター取得関数を追加する
- API 入口を追加する
- 少なくとも以下を返せるようにする
  - ステータス候補
  - ショップ候補
  - 商品担当者候補
  - 配送元候補
  - 配送業者候補
- `doPost` は新規追加しない

## 業務ルール
- 既存コードのハードコード定義と整合すること
- 返却形式は UI から使いやすい構造にすること
- 未定義マスターは安全に空配列で返すこと
- 既存 API 基盤を壊さないこと

## 完了条件
- `20_api_netshop.gs` に実コード差分が入ること
- マスター取得 API 本体が追加されていること
- `md/json` だけの変更で終わらないこと
- 実装ファイル差分が存在すること

## 非対象
- UI 実装
- [18] バリエーション確認API

## 注意
- 調査ではなく実装 Issue
- 1 Issue = 1機能
- 対象外の機能を混ぜないこと
