## 背景
NETSHOP GAS の Web化 Phase1 を進めます。
[15] キャンセル処理API が完了したため、次は [16] 一括削除 / 無効化API を実装対象とします。

## 正式番号
- [16] 一括削除 / 無効化API

## 実装対象ファイル
- `20_api_netshop.gs`

## 目的
複数の商品 / 取引を一括で無効化または削除対象として扱える API を追加する。

## 必須実装
- 一括無効化または削除処理関数を追加する
- API 入口を追加する
- 複数IDを受け取って処理できるようにする
- 成功件数、失敗件数、成功ID一覧、失敗ID一覧を返す
- `doPost` は新規追加しない

## 業務ルール
- 対象シートは `統合売上データ`
- 管理ID は B列
- 物理削除ではなく、まずは安全な無効化寄りの実装を優先する
- 既存データを破壊しないこと
- 存在しないIDは安全に失敗として返すこと
- 既存 API 基盤を壊さないこと

## 完了条件
- `20_api_netshop.gs` に実コード差分が入ること
- 一括削除 / 無効化 API 本体が追加されていること
- `md/json` だけの変更で終わらないこと
- 実装ファイル差分が存在すること

## 非対象
- UI 実装
- 物理削除の強行実装

## 注意
- 調査ではなく実装 Issue
- 1 Issue = 1機能
- 対象外の機能を混ぜないこと
- 安全性優先で実装すること
