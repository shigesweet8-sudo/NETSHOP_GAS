## 背景
NETSHOP GAS の Web化 Phase1 を進めます。
[14] 並び替えAPI が完了したため、次は [15] キャンセル処理API を実装対象とします。

## 正式番号
- [15] キャンセル処理API

## 実装対象ファイル
- `20_api_netshop.gs`

## 目的
指定した商品 / 取引をキャンセル状態へ更新できる API を追加する。

## 必須実装
- キャンセル処理関数を追加する
- API 入口を追加する
- 指定IDのステータスを `キャンセル` に更新できるようにする
- 必要に応じて DATE 更新を行う
- `doPost` は新規追加しない

## 業務ルール
- 対象シートは `統合売上データ`
- 管理ID は B列
- ステータスは A列
- DATE は D列
- 既存のステータス更新ロジックと矛盾しないこと
- 存在しないIDは安全にエラーで返すこと
- 既存 API 基盤を壊さないこと

## 完了条件
- `20_api_netshop.gs` に実コード差分が入ること
- キャンセル処理 API 本体が追加されていること
- `md/json` だけの変更で終わらないこと
- 実装ファイル差分が存在すること

## 非対象
- UI 実装
- [16] 一括削除 / 無効化API

## 注意
- 調査ではなく実装 Issue
- 1 Issue = 1機能
- 対象外の機能を混ぜないこと
