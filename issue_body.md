## 背景
NETSHOP GAS の Web化 Phase1 を進めます。
[17] マスター取得API が完了したため、次は [18] バリエーション確認API を実装対象とします。

## 正式番号
- [18] バリエーション確認API

## 実装対象ファイル
- `20_api_netshop.gs`

## 目的
商品登録・更新時に、入力内容の妥当性を事前確認できる API を追加する。

## 必須実装
- 入力内容の確認関数を追加する
- API 入口を追加する
- 少なくとも以下を確認できるようにする
  - 必須項目の不足
  - ステータス値の妥当性
  - SHOP 値の妥当性
  - 金額項目が数値として扱えるか
- エラー一覧と警告一覧を返せるようにする
- `doPost` は新規追加しない

## 業務ルール
- 既存の createItem / updateItem と整合すること
- 既存マスター値と矛盾しないこと
- 未入力や空文字は安全に扱うこと
- 例外で落とさず、確認結果として返すこと
- 既存 API 基盤を壊さないこと

## 完了条件
- `20_api_netshop.gs` に実コード差分が入ること
- バリエーション確認 API 本体が追加されていること
- `md/json` だけの変更で終わらないこと
- 実装ファイル差分が存在すること

## 非対象
- UI 実装
- [14] 並び替えAPI
- [15] キャンセル処理API

## 注意
- 調査ではなく実装 Issue
- 1 Issue = 1機能
- 対象外の機能を混ぜないこと
